gdb 调试方法

常用流程&命令

gdb 调用gdb调试，进入gdb回话入口

  • gdb 直接进入gdb，然后在gdb功能界面中，使用file加载相关调试对象。
  • gdb target 调试目标程序。
  • gdb target processid 将目标程序附加到进程中调试。
  • gdb target corefile core文件调试，参考链接：

http://blog.csdn.net/ithomer/article/details/5945152 http://100continue.iteye.com/blog/1821893

基本命令(括号里为命令缩写形式)

help command 帮助命令，可以查看相关命令（command）的说明文档。

layout：用于分割窗口，可以一边查看代码，一边测试：

layout src：显示源代码窗口 layout asm：显示反汇编窗口 layout regs：显示源代码/反汇编和CPU寄存器窗口 layout split：显示源代码和反汇编窗口 Ctrl + L：刷新窗口
